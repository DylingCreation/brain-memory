## 五、问题与根因

| 问题 | 根因 | 改进措施 | 对应方法论 |
|------|------|---------|-----------|
| **awk 破坏 context.ts** | 文本操作后未立即 build 验证；git checkout 丢失 F-1 改动 | 每次操作后立即 `npm run build`；优先使用 `write` 而非 `awk`/`sed` | §6.3 分批验证 |
| **10x 性能目标未达标** | 200 节点子图提取开销占主导；1k 节点提升到 4.4x | 10x 目标定位为 10k+ 节点场景 | §8.2 量化验收 |
| **影响面估算不足 ×2** | F-1 只评估 src/ 层（120 行），忽略 test/ 层（16 文件） | 影响分析必须含测试层；预估 ×1.5~2 校准 | §2.2 工作量校准 |
| **LLM 测试不稳定** | mock-openclaw 11 用例串行 40+ 次 API 调用 → 42s + 随机超时 | 记录为已知问题，建议加 `BM_FAST_TEST` 门控 | §6.4 降级设计 |
| **JSDoc 扫描脚本多轮迭代** | 初版只检测单行 JSDoc，忽略多行注释 | 改进脚本支持多行 JSDoc + 空白行跳过 | §7.1 基于实测 |
