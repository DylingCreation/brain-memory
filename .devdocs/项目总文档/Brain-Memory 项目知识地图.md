## 按使用场景导航

### 🆕 我是新接手的开发者

```
1. 先读「模块总览」→ 搞清代码在哪
2. 再读「数据流文档 L3」→ 搞清 6 条业务管道
3. 读「边界与运行」→ 搭本地环境
4. 查「API 契约」→ 改代码时知道契约承诺
```

### 🔧 我要改一个模块

```
1. 查「数据流文档 L1」→ 找到对应模块的内部数据流
2. 查「数据流文档 L2」→ 确认 Service 依赖关系
3. 查「API 契约 → 测试矩阵」→ 确认测试覆盖情况
4. 查「演进记录 → 技术债务」→ 避免踩已知坑
```

### 🚀 我要部署

```
1. 读「边界与运行」→ 了解依赖和运行要求
2. 查项目部署文档 → docs/deployment.md
```

### 📋 我要做版本规划

```
1. 读「演进记录」→ 了解版本节奏和历史决策
2. 查 .devdocs/INDEX.md → 获取过往摸底/规划/复盘全文
```

### 🔍 我要理解某个数据转换

```
1. 查「数据流文档 L0」→ 9 条核心类型变形链
2. 查「数据流文档 L3」→ 端到端管道时序图
```
