## 一、调研目的

分析 `extractor/extract.ts` 的 LLM prompt 模板，识别可被规则/模板覆盖的内容，设计三级提取方案（启发式 → LLM 校验 → 多轮合并），降低 LLM 不可用时提取功能完全跳过的问题。

## 二、步骤 1：LLM prompt 逐项分析

### 2.1 LLM 调用全景

| 位置 | 调用 | 用途 |
|------|------|------|
| `extract()` 第 131 行 | `this.llm(EXTRACT_SYS, userPrompt)` | 知识提取（主路径） |
| `extract()` 第 134 行 | `this.llm(EXTRACT_SYS, fixPrompt)` | JSON 解析失败重试 |
| `finalize()` 第 158 行 | `this.llm(FINALIZE_SYS, ...)` | session 结束前审查 |

### 2.2 EXTRACT_SYS prompt 5 个提取维度评估

| # | 提取维度 | 可规则化？ | 评估 |
|---|---------|-----------|------|
| ① 节点类型判断 | ⚠️ 部分 | 事件识别可做规则（报错关键词+代码块），但"可复用技能"需语义理解 |
| ② 记忆分类（8 类） | ⚠️ 部分 | 偏好/实体/任务可做规则，profile/cases/patterns 需上下文 |
| ③ 边关系提取（11 种） | ✅ 可规则化 | 本质是结构化规则：已知节点类型，按决策树选择边类型 |
| ④ 噪声过滤 | ✅ 已规则化 | `src/noise/filter.ts` 已实现 |
| ⑤ 角色策略 | ⚠️ 部分 | 代码块/命令可规则提取，意图/偏好需语义理解 |

### 2.3 结论

- **硬规则（~30%）**：噪声过滤（已实现）、边关系决策树、代码块/命令提取、工具名/版本号/路径识别
- **软理解（~70%）**：节点类型判断、记忆分类、用户意图/偏好理解、案例/模式抽象

启发式提取的切入点不是完全替代 LLM，而是实现一个**粗提取层**。

## 四、步骤 3：三级提取架构设计

### 4.1 三级架构

```
┌─────────────────────────────────────────────────┐
│ 一级：启发式提取（纯规则，毫秒级）                  │
│ - 8类正则模式匹配 → 产出粗粒度节点                   │
│ - 代码块/命令直接提取 → 产出技能节点                  │
│ - 无需 LLM，延迟 < 10ms                           │
├─────────────────────────────────────────────────┤
│ 二级：LLM 校验（LLM 可用时补充）                     │
│ - 启发式结果作为 baseline 传给 LLM                   │
│ - LLM 仅做"补充+修正"，不复述已提取内容                │
│ - prompt 缩小 50-70%，调用成本大幅下降                │
├─────────────────────────────────────────────────┤
│ 三级：多轮合并（复用现有 compactTurnCount=6）          │
│ - 合并启发式+LLM 结果，去重                            │
└─────────────────────────────────────────────────┘
```

### 4.2 置信度判断逻辑

```
启发式产出节点数 ≥ 3 且有至少 2 类记忆 → 置信度高 → 二级（LLM 补充）
启发式产出节点数 1-2 或仅 1 类记忆 → 置信度中 → 三级（纯 LLM）
启发式产出为空 → 置信度低 → 三级（纯 LLM）
```

### 4.3 降级效果矩阵

| 功能 | LLM 可用（当前） | LLM 不可用（当前） | LLM 不可用（A-2 目标） |
|------|----------------|-------------------|---------------------|
| 提取（extract） | ✅ 完整提取 | ⏭️ 跳过，返回空 | ✅ 粗粒度提取（启发式） |
| 轮次反思 | ✅ 完整反思 | ⏭️ 跳过 | ⏭️ 跳过（依赖 LLM） |
| 知识融合 | ✅ LLM 决策 | ⚠️ 启发式自动合并 | ⚠️ 启发式自动合并 |
| 会话反思 | ✅ 完整反思 | ⏭️ 跳过 | ⏭️ 跳过 |
| 推理 | ✅ 完整推理 | ⏭️ 跳过 | ⏭️ 跳过 |
| 召回 | ✅ 正常 | ✅ 正常 | ✅ 正常 |
| 工作记忆 | ✅ 正常 | ✅ 正常 | ✅ 正常 |

| 维度 | LLM 基线 | 启发式目标 |
|------|---------|-----------|

|------|--------------|-------------------|

## 六、风险与预案

| 风险 | 等级 | 预案 |
|------|------|------|
| 启发式提取质量不及 LLM（有效率 < 50%） | 中 | 保留 LLM 校验兜底，不强制关闭 LLM 路径 |
| 正则规则对非典型对话匹配率低 | 中 | 增加兜底规则，扩大正则覆盖范围 |
| 三级决策流增加代码复杂度 | 低 | 分层清晰，每个层级独立可测 |
