## 三、逐项验证结果

### 3.1 参数化查询（82 处）✅ 全部安全

所有使用 `.prepare()` 的查询，均通过 `?` 占位符绑定参数，无字符串拼接用户输入到 SQL 中。

| 查询类型 | 数量 | 示例 |
|---------|------|------|
| 带参数的单值查询 | 40+ | `WHERE id=?`、`WHERE name=?`、`WHERE session_id=?` |
| 批量参数查询 | 15+ | `WHERE id IN (?,?,?)` + `.all(...ids)` |
| 多参数 INSERT/UPDATE | 20+ | `INSERT ... VALUES (?,?,?,?,?)`、`UPDATE ... SET col1=?, col2=?` |
| 无参数聚合查询 | 7 | `SELECT COUNT(*) FROM bm_nodes` |

### 3.2 模板字符串 SQL — 安全性验证 ✅

部分查询使用模板字符串构造 SQL，需验证拼接内容是否安全：

| 文件 | 位置 | 拼接内容 | 安全性 | 说明 |
|------|------|---------|--------|------|
| `store.ts` | searchNodes | `${ftsQuery}` — FTS5 MATCH 参数 | ✅ 安全 | ftsQuery 通过 `.all(ftsQuery, ...)` 作为参数传入，非 SQL 拼接 |
| `store.ts` | searchNodes | `${where}` — LIKE 条件链 | ✅ 安全 | 仅包含 `name LIKE ?` 占位符结构，值通过参数传递 |
| `store.ts` | allActiveNodes | `${clause}` — scope 过滤 | ✅ 安全 | `buildScopeFilterClause()` 返回的参数化子句 |
| `store.ts` | vectorSearchWithScore | `${placeholders}` — IN 列表 | ✅ 安全 | 由 `nodes.map(() => "?").join(",")` 生成，仅含 `?` |
| `store.ts` | graphWalk | `${ph}` — 递归 CTE | ✅ 安全 | 由 `seedIds.map(() => "?").join(",")` 生成，仅含 `?` |
| `store.ts` | nodesByCommunityIds | `${ph}` — IN 列表 | ✅ 安全 | 由 `communityIds.map(() => "?").join(",")` 生成，仅含 `?` |

### 3.3 db.exec() 使用 — 安全性验证 ✅

| 文件 | SQL 内容 | 用途 | 安全性 |
|------|---------|------|--------|
| `store/db.ts` | `SCHEMA`（完整建表 DDL） | 数据库初始化 | ✅ 纯 DDL，无用户输入 |
| `store/migrate.ts` | `META_TABLE`（bm_meta 建表） | 迁移元数据表创建 | ✅ 纯 DDL，无用户输入 |
| `store/store.ts` | `BEGIN` / `COMMIT` / `ROLLBACK` | 事务控制 | ✅ 纯事务命令，无用户输入 |

### 3.4 Scope 隔离参数化验证 ✅

`src/scope/isolation.ts` 中 `buildScopeFilterClause()` 函数生成的 SQL 子句：
- 仅拼接 `scope_session = ?`、`scope_agent = ?` 等占位符结构
- 实际值存储在返回的 `params` 数组中
- 调用方通过 `.all(...params)` 传递参数

### 3.5 FTS5 MATCH 验证 ✅

`searchNodes` 中 FTS5 全文检索：
```typescript
const ftsQuery = terms.map(t => `"${t.replace(/"/g, "")}"`).join(" OR ");
const rows = db.prepare(`... WHERE bm_nodes_fts MATCH ? ...`).all(ftsQuery, ...);
```
- `ftsQuery` 作为 MATCH 的参数值，通过 `?` 占位符传递
- 不是 SQL 字符串拼接，是参数绑定

## 五、安全建议

1. **代码审查时专项检查**：未来新增 SQL 查询时，在 code review 中增加「是否使用 `?` 占位符」检查项
2. **禁止动态拼接 SQL**：禁止使用字符串拼接或模板变量直接构造 SQL，如需动态 SQL，必须使用参数化查询
3. **定期审计**：每次大版本更新前，重新运行 SQL 参数化审计
