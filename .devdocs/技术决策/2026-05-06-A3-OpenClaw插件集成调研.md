### 1.3 registerHook 事件体系

**事件格式**：`registerHook` 接受 `"eventType:action"` 格式的字符串或字符串数组。

**内部事件类型**（`InternalHookEventType`）：`"command" | "session" | "agent" | "gateway" | "message"`

**Handler 签名**：
```typescript
type InternalHookHandler = (event: InternalHookEvent) => Promise<void> | void;

interface InternalHookEvent {
  type: InternalHookEventType;    // "command" | "session" | "agent" | "gateway" | "message"
  action: string;                  // 具体动作名
  sessionKey: string;              // 会话标识
  context: Record<string, unknown>; // 额外上下文
  timestamp: Date;
  messages: string[];              // 可追加回复消息
}
```

**已确认触发的内部事件**（来自 `internal-hooks-Cv9ukygq.js`）：

| 事件 | 格式 | 说明 |
|------|------|------|
| `agent:bootstrap` | agent:bootstrap | Agent 启动 |
| `gateway:startup` | gateway:startup | 网关启动 |
| `message:received` | message:received | 用户消息到达 |
| `message:sent` | message:sent | Agent 消息发送 |
| `message:transcribed` | message:transcribed | 语音转录 |
| `message:preprocessed` | message:preprocessed | 消息预处理 |
| `session:patch` | session:patch | 会话修补 |

## 二、当前 brain-memory 实现与真实 API 的差异

### 2.1 钩子注册方式（🔴 核心差异）

| 维度 | 当前实现 | 真实 API | 差异 |
|------|---------|---------|------|
| **注册方法** | `api.on('message_received', handler)` | `api.registerHook('message_received', handler)` | ❌ 方法名不同 |
| **事件名称** | `message_received`, `message_sent`, `before_message_write`, `session_start`, `session_end` | 同名 | ✅ 名称正确，在 PLUGIN_HOOK_NAMES 列表中 |
| **Handler 签名** | `(event: any, ctx: any)` | `(event: InternalHookEvent) => void` | ⚠️ 签名不同但可适配 |

### 2.2 插件入口格式（🟡 中等差异）

| 维度 | 当前实现 | 真实 API | 差异 |
|------|---------|---------|------|
| **入口方式** | 直接导出 `register/init/activate/deactivate` 函数 | `definePluginEntry({ register })` | ⚠️ 当前方式可能兼容但非规范 |
| **元数据** | `register()` 返回 `{id, name, version, ...}` | `definePluginEntry` 参数中声明 | ⚠️ 结构不同 |

### 2.3 openclaw.plugin.json（🟡 中等差异）

当前 `hooks` 列表使用旧式函数名（`handleMessage`, `onSessionStart`, `beforeMessageSend`），与真实 Plugin Hook 名称不匹配。

### 2.4 setup-openclaw.js（🟢 轻微差异）

- 交互式配置，但预设方案不够完善
- 默认 Embedding 配置为本地 Ollama（`bge-m3:localhost:11434`），不通用
- 已有备份机制，结构基本正确

## 三、开发实施计划

### 步骤 1：创建新的插件入口文件

新建 `openclaw-plugin-entry.ts`，使用 `definePluginEntry()` 规范格式：

```typescript
import { definePluginEntry } from 'openclaw/plugin-sdk/core';

export default definePluginEntry({
  id: 'brain-memory',
  name: 'Brain Memory',
  description: 'Unified knowledge graph + vector memory system',
  register: (api) => {
    api.registerHook('message_received', message_received_handler);
    api.registerHook('message_sent', message_sent_handler);
    api.registerHook('before_message_write', before_message_write_handler);
    api.registerHook('session_start', session_start_handler);
    api.registerHook('session_end', session_end_handler);
  },
});
```

### 步骤 2：适配 Hook Handler 签名

将当前 `(event: any, ctx: any)` 签名适配为 `(event: InternalHookEvent) => void`：

- `event.context` 中提取 `conversationId`, `accountId`, `channelId` 等字段
- `event.sessionKey` 替代 `ctx.conversationId`
- `event.messages.push()` 用于回复用户

### 步骤 3：更新 setup-openclaw.js

- 增加预设方案（DashScope/OpenAI/SiliconFlow/自定义）
- 修正默认 Embedding 配置

### 步骤 4：更新 openclaw.plugin.json

- hooks 列表与真实 Plugin Hook 名称对齐

### 步骤 5：集成验证

- 在真实 OpenClaw 环境中测试 hook 触发

## 五、结论

**核心发现**：brain-memory 的 Hook 名称（`message_received`, `message_sent`, `before_message_write`, `session_start`, `session_end`）**全部正确**，在 OpenClaw 的 PLUGIN_HOOK_NAMES 列表中。只需修改注册方法从 `api.on()` 改为 `api.registerHook()`，并适配 Handler 签名即可。
