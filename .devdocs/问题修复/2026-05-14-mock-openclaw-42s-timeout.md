# mock-openclaw 集成测试 42 秒超时问题

| 字段 | 内容 |
|------|------|
| **日期** | 2026-05-14 |
| **发现人** | 代码助手 |
| **关联版本** | v1.2.0 |
| **严重度** | 🟡 中（不影响功能，影响 CI/开发体验） |

---

## 现象

`test/mock-openclaw-integration.test.ts` 全部 11 个用例总耗时 **42.7 秒**，相对其他测试（10-30ms）高出 1000+ 倍。

## 根因

该测试做了完整的 LLM 调用链路（extract + recall + reflection + maintenance），每个步骤都调用真实 DashScope API。11 个用例，每个有 3-5 次 LLM 调用，累计约 40 次 API 调用。

## 影响

- 开发体验差（每次全量跑等 43 秒）
- 可能触发 API rate limit
- CI 环境无 API Key 时直接 skip（当前状态）

## 建议方案

| 方案 | 描述 | 工作量 |
|------|------|--------|
| A | 缩短 LLM 调用次数（合并相邻用例、减少冗余测试） | 0.5 天 |
| B | 添加 `describe.skipIf(!BM_FAST_TEST)` 快速模式门控 | 0.25 天 |
| C | Mock LLM 响应（部分用例用 mock，只保留 1-2 个真实调用） | 1 天 |
| **推荐** | **B** — 最小改动，快速见效 | 0.25 天 |
