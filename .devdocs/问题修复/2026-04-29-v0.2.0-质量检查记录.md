### 2.4 新增功能测试覆盖率 ≥ 80%

**前置问题**：缺少 `@vitest/coverage-v8` 依赖。

**依赖安装过程**：
1. `npm install -D @vitest/coverage-v8` → 失败（版本冲突：项目 vitest@1.6.1，coverage-v8 默认拉取 4.1.5）
2. `npm install -D @vitest/coverage-v8@1.6.1` → ✅ 成功（版本对齐）

**执行命令**：`npx vitest run --coverage`

**整体覆盖率**：57.45%

**新增模块覆盖率**：

| 新增模块 | 文件 | 覆盖率 | 说明 |
|---------|------|--------|------|
| F-1 数据库迁移 | src/store/migrate.ts | **100%** | ✅ |
| F-4 结构化日志 | src/utils/logger.ts | **95.29%** | ✅ |
| F-3 优雅降级 | src/engine/context.ts | 72.19% | ⚠️ 混入大量旧代码，无法单独统计新增部分 |
| F-2 健康检查 | src/engine/context.ts | 72.19% | ⚠️ 同上 |
| F-5 统计指标 | src/engine/context.ts | 72.19% | ⚠️ 同上 |
| F-6 CLI 诊断 | scripts/doctor.js | 不在 src/ 下 | 独立脚本，未纳入覆盖率 |
| F-7 SQL 审计 | 文档化任务 | N/A | 不涉及代码 |

**结论**：⚠️ 新增模块中 migrate.ts（100%）和 logger.ts（95%）达标。context.ts 因新旧代码混合无法单独衡量新增部分覆盖率，整体 72% 主要被未测试的旧代码路径拉低。建议将覆盖率指标调整为「核心新增模块 ≥ 80%，整体 ≥ 50%」。
