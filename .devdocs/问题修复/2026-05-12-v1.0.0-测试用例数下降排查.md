## 五、经验教训

| 教训 | 改进措施 |
|------|---------|
| 开发记录写了但代码未提交 | **留痕必须与代码提交同步**：开发记录中的「测试全通过」必须对应一个已推送的 git commit |
| 留痕文档中的数字未与仓库交叉验证 | 后续摸底时，留痕记录的测试数必须与 `git ls-tree` 和 `vitest run` 实测交叉验证 |
| 446 用例的数字被当作基线引用 | 基线数据必须以仓库实测为准，不能仅依赖留痕文档 |
