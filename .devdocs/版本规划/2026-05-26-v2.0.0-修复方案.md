## 任务 1.2：修复 scope_id 迁移算法的不一致 🔴 P0

**来源**: M3-P0-1  
**预计耗时**: 1-2 小时  
**影响**: 所有已运行 v2 迁移的数据库，scope 隔离功能已损坏

### 问题简述

`migrate.ts` 使用 `lower(hex(CONCAT(...)))` 生成 scope_id，而 `isolation.ts` 使用 `sha256().hex.slice(0,16)`。两种算法产生完全不同的 scope_id，导致迁移后的旧节点在 scope 过滤时永远无法被匹配到。

### 修复方案

**选择方案 B（逐行 JS 修正）**——不依赖 SQLite 扩展加载：

```typescript
// 文件: src/store/migrate.ts

// 修改 migrateToV2_ScopeUpgrade 函数：

import { createHash } from 'crypto';

function migrateToV2_ScopeUpgrade(db: DatabaseSyncInstance): void {
  // ... 现有的 ALTER TABLE 和 ADD COLUMN 语句保持不变 ...
  
  // 🔴 删除：原有的 SQL UPDATE 使用 hex(CONCAT(...)) 的语句
  
  // ✅ 新增：逐行用 JS 计算正确的 scope_id
  const computeScopeId = (platform: string, workspace: string, 
                          agent: string, user: string, 
                          chat: string, thread: string): string => {
    const key = [platform, workspace, agent, user, chat, thread]
      .map(v => v || '')
      .join(':');
    return createHash('sha256').update(key).digest('hex').slice(0, 16);
  };

  // 查询所有需要更新的行
  const rows = db.prepare(`
    SELECT id, scopePlatform, scopeWorkspace, scopeAgent, 
           scopeUser, scopeChat, scopeThread
    FROM bm_nodes WHERE scope_id IS NULL OR scope_id = ''
  `).all() as any[];

  // 逐行更新（使用预编译语句避免 SQL 注入）
  const update = db.prepare('UPDATE bm_nodes SET scope_id = ? WHERE id = ?');
  
  const tx = db.transaction(() => {
    for (const row of rows) {
      const scopeId = computeScopeId(
        row.scopePlatform, row.scopeWorkspace, row.scopeAgent,
        row.scopeUser, row.scopeChat, row.scopeThread
      );
      update.run(scopeId, row.id);
    }
  });
  tx();
  
  logger.info(`[Migration] Computed scope_id for ${rows.length} existing nodes`);
}
```

### 验证

1. 创建测试数据库，插入若干节点
2. 运行迁移
3. 用 `computeScopeId()` (JS) 计算期望值
4. 查询数据库 `SELECT scope_id FROM bm_nodes`，逐行比对

```bash
npm test -- src/store/migrate.test.ts
```

### 附加修复

将此迁移 idempotent 化：如果 `scope_id` 已经存在且不为空，跳过该行（而非覆盖）。防止重复迁移产生再次不一致。

// 1. 增强 src/utils/tokens.ts 的 estimateNodeTokens
//    合并 vector-recall.ts 的中英文感知逻辑
export function estimateNodeTokens(node: { name?: string; content?: string; description?: string }): number {
  const text = [node.name, node.description, node.content].filter(Boolean).join(' ');
  return estimateTokens(text);
}

export function estimateTokens(text: string): number {
  // 合并原 vector-recall.ts 的中英文感知算法
  let charCount = 0;
  let chineseCount = 0;
  for (const ch of text) {
    const code = ch.charCodeAt(0);
    if (code >= 0x4e00 && code <= 0x9fff) {
      chineseCount++;
    } else if (code >= 0x3000 && code <= 0x303f) {
      chineseCount++;
    } else {
      charCount++;
    }
  }
  return Math.ceil(chineseCount / 1.9 + charCount / 3.8);
}

## 任务 3.3：消除 92 处非确定性测试模式 🟡 P2

**来源**: M9-P2-1

### 分类处理

| 非确定性来源 | 数量（估计） | 修复手段 |
|-------------|------------|----------|
| `Math.random()` | ~25 处 | `vi.spyOn(Math, 'random').mockReturnValue(0.5)` |
