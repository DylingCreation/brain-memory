## 三、问题清单与修复方案

### P0 — 功能 Bug（3 项，必须修复）

| # | 问题 | 文件 | 修复方案 |
|---|------|------|---------|
| 1 | 硬编码 Linux `/tmp/` | `test/c7-recall-baseline.test.ts` | `import { tmpdir } from "os"; join(tmpdir(), ...)` |
| 2 | LPA 阈值过紧 | `test/perf-1k-benchmark.test.ts` | ≥ 2x → ≥ 1.5x，或改用相对判定 |
| 3 | dotenv 不一致 | `test/llm-integration-real.test.ts` | 统一在 `vitest.config.ts` 配 `envDir` |

### P1 — 跨平台 + 文档（4 项）

| # | 问题 | 修复方案 |
|---|------|---------|
| 4 | CHANGELOG 数据不符 | 标注"含 LLM"/"不含 LLM"两组数据 |
| 5 | v1.3.0 未报告覆盖率 | 补报告 + 后续版本强制包含 |
| 6 | bash 脚本 Windows 不兼容 | `scripts/release.sh` → `scripts/release.mjs`（Node.js 重写） |
| 7 | vitest 线程池 Windows 文件锁 | `pool: 'threads'` → `pool: 'forks'` 或单线程 |

### P2 — 安全 + 工程（4 项）

| # | 问题 | 修复方案 |
|---|------|---------|
| 8 | 5 个 moderate 漏洞 | `npm audit fix`（可修的修，不可修的记录） |
| 9 | 6 个 deprecated 依赖 | rimraf 3→5，glob 7→10（如兼容） |
| 10 | CI/CD 缺失 | `.github/workflows/test.yml`（Node 18/20/22 矩阵） |
| 11 | npm 发布不完整 | `files` 字段补 `src/` + `test/`，npm publish 1.1.0~1.3.0 |
