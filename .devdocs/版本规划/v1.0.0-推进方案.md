## 十、验收闭环

### 10.1 验收标准溯源（方法论 §8）

| 原则 | 本方案体现 |
|------|-----------|
| **可量化** | 每项功能有明确验收标准（如「覆盖率 ≥ 70%」而非「覆盖率提升」） |
| **可验证** | 验收标准对应具体命令（`npm test`、`vitest run --coverage` 等） |
| **可追溯** | 每次验收结果写入留痕文档 |
| **无空白** | 即使是文档更新也有验收标准（「已更新」） |

### 10.2 验收流程

```
功能开发完成
    ↓
代码助手自检（npm test + build + tsc）
    ↓
输出批次验收报告
    ↓
老板端到端验收
    ↓
老板确认 → 进入下一批次
    ↓
（若发现问题 → 修复 → 重新验证）
```

### 10.3 版本验收

全部 4 个批次通过后，执行：

1. 全维度摸底（构建/测试/覆盖率/性能/Git/依赖/文档）
2. 发布检查清单逐项验证
3. 输出版本验收报告
4. 老板确认后发布
5. 发布后 48h 内完成版本复盘
