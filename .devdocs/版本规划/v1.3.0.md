## 三、新增功能

### F-13：LanceDB POC（🟡 P1）

**目的**：验证 LanceDB 作为向量存储后端的可行性，不承诺生产级。

**调研结论**：
- LanceDB 原生支持 ANN 近似最近邻搜索，比 SQLite BLOB 余弦搜索更快
- IStorageAdapter 接口已就绪，只需新实现
- 图算法（PageRank/LPA）需在内存中执行（LanceDB 不原生支持图）
- 官方 TypeScript SDK 可用

**实施步骤**：

| 步骤 | 内容 | 产出 |
|------|------|------|
| 1 | 安装 `@lancedb/lancedb`，验证本地运行 | 安装验证报告 |
| 2 | 设计 LanceDB Table schema（节点表 + 边表） | 设计文档 |
| 3 | 实现 `LanceDBStorageAdapter`（CRUD + 向量搜索） | 代码 |
| 4 | 向量搜索性能对比（LanceDB vs SQLite） | 对比报告 |
| 5 | 编写测试（最小集：连接 + CRUD + 搜索） | 测试 |

**验收标准**：
- LanceDB Table 能存储和查询节点
- ANN 搜索比 SQLite 余弦搜索快（至少 2x @ 1k 节点）
- 不破坏现有 SQLite 路径

### F-15：npm 包优化（🟢 P3）

**现状**：`npm pack` 203.3 kB，211 文件，包含 `scripts/` 目录（120+ 文件）。

**问题**：
- `scripts/` 是开发工具，npm 用户不需要
- `docs/api/`（typedoc 生成的 120 文件）体积大，用户可通过其他方式访问
- `package.json` 的 `files` 字段只包含 `dist/**/*`，但 npm 默认包含不在 `.npmignore` 中的文件

**目标**：
- 精简 npm 包体积 ≤ 50%
- `scripts/` 不入库
- `docs/api/` 不入库

**实施步骤**：

| 步骤 | 内容 |
|------|------|
| 1 | 审查现有 `.npmignore` |
| 2 | 添加 `scripts/`、`docs/api/` 等屏蔽 |
| 3 | 验证 `npm pack --dry-run` 输出 |

### F-16：性能基准验证（🟡 P1）

**背景**：v1.1.0 增量维护在 200 节点下 2.7x，未达到 10x 目标。

**目标**：验证 1k+ 节点下增量 vs 全量 ≥ 5x。

**实施步骤**：

| 步骤 | 内容 |
|------|------|
| 1 | 编写性能基准测试：生成 1k 节点图 |
| 2 | 对比全量 vs 增量 PageRank 耗时 |

| 4 | 记录结果并写入性能基准文件 |
