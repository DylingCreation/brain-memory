## 二、调研：LanceDB 可行性

### 2.1 LanceDB 简介

LanceDB 是一个开源嵌入式向量数据库，支持：
- 持久化存储（本地磁盘）
- ANN 近似最近邻搜索（更快）
- 过滤 + 向量混合查询
- Python/TypeScript/Node.js SDK

### 2.2 与 brain-memory 的适配度分析

| 维度 | 分析 | 判定 |
|------|------|------|
| 向量存储 | LanceDB 原生支持，比 SQLite BLOB 更高效 | ✅ 优势 |
| CRUD 抽象 | IStorageAdapter 已定义，只需新实现 | ✅ 已就绪 |
| 图谱能力 | LanceDB 不原生支持图算法（PageRank/LPA） | ⚠️ 需自建 |
| 本地部署 | 支持，无需外部服务 | ✅ 符合定位 |
| 依赖复杂度 | 新增 `@lancedb/lancedb` 包 | ⚠️ 需验证 |
| TypeScript 支持 | 官方 SDK 支持 | ✅ |

### 2.3 POC 方案

```
LanceDBStorageAdapter 实现 IStorageAdapter:
├── 向量操作 → LanceDB 原生 API
├── 节点/边 CRUD → LanceDB Table（或混合：SQLite + LanceDB）
├── 搜索 → LanceDB ANN search
└── 图算法（PageRank/LPA） → 内存中执行（加载全表）
```

**建议**：先做可行性验证（POC），验证：
1. LanceDB Table 能否存储节点/边数据
2. ANN 搜索性能对比 SQLite
3. 与 IStorageAdapter 接口的兼容性

**不承诺**：v1.3.0 只做 POC，不保证生产级。
