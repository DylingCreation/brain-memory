## 三、依赖更新详情

| 包 | 当前版本 | 最新可用 | 说明 |
|------|---------|---------|------|
| `@photostructure/sqlite` | 1.2.0 | 1.2.1 | 补丁版本，可安全升级（v0.1.9 摸底时已建议） |
| `@types/node` | 20.19.39 | 25.6.0 | 大版本跳跃，不建议升 |
| `@vitest/coverage-v8` | 1.6.1 | 4.1.5 | 需与 vitest 版本对齐 |
| `eslint` | 8.57.1 | 10.3.0 | 大版本跳跃 |
| `typedoc` | 0.25.13 | 0.28.19 | 文档生成工具 |
| `typescript` | 5.4.5 | 6.0.3 | 大版本跳跃，升级风险高 |
| `vitest` | 1.6.1 | 4.1.5 | 大版本跳跃，升级可能破坏测试 |

**结论**：关键建议是升级 `@photostructure/sqlite` 1.2.0→1.2.1（从 v0.1.9 就已建议但未执行）。其余大版本升级不在 v1.0.0 范围内，后续版本考虑。

**npm audit**：5 个 moderate 漏洞，全部来自开发依赖链（vitest → vite → esbuild），非 brain-memory 自身运行时依赖。

| 漏洞 | 严重等级 | 来源 | 说明 |
|------|---------|------|------|
| esbuild ≤0.24.2 dev server 请求泄露 | moderate | esbuild | 开发时任意网站可发送请求到 dev server 并读取响应 |
| vite ≤6.4.1 依赖脆弱 esbuild | moderate | vite | 传递依赖 |
| vite-node ≤2.2.0-beta.2 依赖脆弱 vite | moderate | vite-node | 传递依赖 |
| vitest 0.0.1~2.2.0-beta.2 依赖脆弱 vite | moderate | vitest | 传递依赖 |
| @vitest/coverage-v8 ≤2.2.0-beta.2 依赖脆弱 vitest | moderate | @vitest/coverage-v8 | 传递依赖 |

**处置建议**：全部为开发时漏洞，不影响 npm 包用户。`npm audit fix --force` 会将 vitest 升至 4.1.5（破坏性变更），需在后续大版本升级时处理，**不在 v1.0.0 范围内**。
