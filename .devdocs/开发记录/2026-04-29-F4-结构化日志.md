## 一、开发内容

### 1. 新增文件

#### `src/utils/logger.ts` — 结构化日志核心模块

| 组件 | 说明 |
|------|------|
| `LogLevel` 类型 | `"error" \| "warn" \| "info" \| "debug"` |
| `LEVEL_PRIORITY` | 日志级别优先级映射（error=0 < warn=1 < info=2 < debug=3） |
| `getMinLevel()` | 从 `BM_LOG_LEVEL` 环境变量读取最小日志级别，默认 `info` |
| `formatTime()` | 时间戳格式 `YYYY-MM-DD HH:MM:SS.mmm` |
| `levelLabel()` | 级别标签格式化（ERROR/WARN/INFO/DEBUG，固定宽度对齐） |
| `emit()` | 核心输出函数，按级别过滤后通过 `console.error/warn/log` 输出 |
| `logger` 对象 | 公开 API：`error()`, `warn()`, `info()`, `debug()`, `isEnabled()` |

**输出格式**：
```
[brain-memory][2026-04-29 11:45:23.456][INFO ][context] Initialized with 42 existing nodes
```

**环境变量控制**：
| BM_LOG_LEVEL | 可见级别 | 说明 |
|-------------|---------|------|
| `error` | 仅 ERROR | 生产环境，仅看错误 |
| `warn` | ERROR + WARN | 日常运行，关注警告 |
| `info`（默认） | ERROR + WARN + INFO | 默认级别，基本信息 |
| `debug` | 全部 | 调试模式，包含所有调试信息 |

### 3. 兼容层处理

| 旧机制 | 新机制 | 兼容策略 |
|--------|--------|---------|
| `BM_LOG_LLM=1` 门控 LLM 日志 | `logger.debug("llm", ...)` | 保留 `BM_LOG_LLM` 门控，改用 logger 输出 |
| `BM_DEBUG=1` 门控调试日志 | `logger.debug()` | 去除 `BM_DEBUG` 门控，由 `BM_LOG_LEVEL=debug` 统一控制 |
| `[brain-memory]` 前缀 | `[brain-memory][时间][级别][模块]` | 保留 `[brain-memory]` 标识，增强结构化 |

## 四、使用示例

### 默认模式（info 级别）
```bash
# 仅看到 error + warn + info
node index.js
```

### 调试模式
```bash
BM_LOG_LEVEL=debug node index.js
# 看到所有级别的日志，包含 recall 统计、reflect 详情等
```

### 生产模式（仅错误）
```bash
BM_LOG_LEVEL=error node index.js
# 仅看到 error
```

### LLM 请求跟踪（兼容旧机制）
```bash
BM_LOG_LLM=1 node index.js
# LLM 请求/响应/重试日志通过 logger 输出
```

## 五、后续优化建议

1. **异步日志**：当前为同步 `console.*` 输出，高吞吐场景可考虑异步日志
2. **日志文件输出**：可增加 `BM_LOG_FILE` 环境变量支持，输出到文件
3. **彩色输出**：终端环境下可考虑添加 ANSI 彩色区分级别
4. **BM_DEBUG 迁移说明**：在 CHANGELOG 中说明 `BM_DEBUG` 已合并到 `BM_LOG_LEVEL=debug`
