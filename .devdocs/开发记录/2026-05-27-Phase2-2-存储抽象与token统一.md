## 任务 2.3：IStorageAdapter 能力标志位 ✅

### 接口扩展

在 `src/store/adapter.ts` 中新增 `StorageCapabilities` 接口：
```typescript
export interface StorageCapabilities {
  communities: boolean;   // 社区检测
  messages: boolean;      // 消息历史
  vector: boolean;        // 向量存储
  ftsSearch: boolean;     // 全文搜索
  graphTraversal: boolean; // 图遍历
  reflections: boolean;   // 反思存储
}
```

`IStorageAdapter` 增加 `readonly capabilities: StorageCapabilities` 属性。

### 适配器实现

| 能力 | SQLite | LanceDB |
|------|--------|---------|
| communities | ✅ true | ❌ false |
| messages | ✅ true | ❌ false |
| vector | ✅ true | ✅ true |
| ftsSearch | ✅ true | ❌ false |
| graphTraversal | ✅ true | ✅ true |
| reflections | ✅ true | ❌ false |

### LanceDB stub 警告

11 个 stub 方法全部添加 `logger.warn("[LanceDB] xxx not implemented")`：
- 社区功能（8个）：`upsertCommunity`, `getCommunity`, `getAllCommunities`, `pruneCommunities`, `communityVectorSearch`, `findNodesByCommunities`, `findCommunityPeers`, `findCommunityRepresentatives`
- 消息功能（4个）：`saveMessage`, `getUnextractedMessages`, `markMessagesExtracted`, `getEpisodicMessages`

### ContextEngine 后端选择

修复了 `config.storage` 被完全忽略的问题：
```typescript
// 修复前（M7-P1-1）：总是创建 SQLiteAdapter
this.storage = new SQLiteStorageAdapter(config.dbPath);

// 修复后：
if (config.storage === 'lancedb') {
  logger.warn('LanceDB backend selected — limited functionality...');
  this.storage = new LanceDBStorageAdapter(config.dbPath);
} else {
  this.storage = new SQLiteStorageAdapter(config.dbPath);
}
```

### ContextEngine 能力降级

- `runMaintenance()`：检测 `capabilities.communities` → 跳过社区检测
- `reflectOnSession()`：检测 `capabilities.reflections` → 返回空数组

## 任务 2.7：统一 token 估算 ✅

### 修改文件

| 文件 | 修改前 | 修改后 |
|------|--------|--------|
| `retriever/vector-recall.ts` | 内联 10 行 `estimateTokens()`（自实现 chineseRatio + charsPerToken） | 导入并委托到 `estimateNodeTokens(n)` |

### 统一前 vs 统一后

```

  hybrid-recall.ts → content.length — 无感知，最粗糙

统一后: 1 种统一方法
  recall.ts → estimateNodeTokens ✅
  vector-recall.ts → estimateNodeTokens ✅
  hybrid-recall.ts → estimateNodeTokens ✅
```
