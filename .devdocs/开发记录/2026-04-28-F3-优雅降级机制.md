## 一、开发内容

### 核心问题

旧版 `ContextEngine` 在 LLM 未配置时会创建 **mock LLM**（返回 `"Mock response - LLM not configured"`），导致提取产生无效节点、反思产生垃圾数据。

**修复策略**：彻底移除 mock LLM，引入 `llmEnabled` 标志，LLM 不可用时跳过所有依赖步骤而非产生无效数据。

### 1. 修改文件

#### `src/engine/context.ts` — ContextEngine 优雅降级

| 改动 | 说明 |
|------|------|
| 新增 `private llmEnabled: boolean` 字段 | 标记 LLM 是否可用 |
| 移除 mock LLM 逻辑 | `createCompleteFn` 返回 `null` 时不再创建 mock，直接记录 `llmEnabled = false` |
| 构造函数 LLM 初始化 | 从 `throw Error` 改为 `console.warn` + 继续初始化 |
| `processTurn` 轮次反思 | 新增 `this.llmEnabled &&` 守卫，LLM 不可用时跳过 |
| `performFusion` | `createCompleteFn(this.config.llm)!` → `this.llmEnabled ? createCompleteFn(...) : null` |
| `reflectOnSession` | 新增 `if (!this.llmEnabled) return []` 守卫 |
| `performReasoning` | 新增 `if (!this.llmEnabled) return []` 守卫 |

**降级效果矩阵**：

| 功能 | LLM 可用 | LLM 不可用 |
|------|---------|-----------|
| 提取（extract） | ✅ 正常提取 | ⏭️ 跳过，返回空 |
| 轮次反思 | ✅ 正常反思 | ⏭️ 跳过，返回空 |
| 知识融合 | ✅ LLM 决策 | ⚠️ 启发式自动合并/链接 |
| 会话反思 | ✅ 正常反思 | ⏭️ 跳过，返回空 |
| 推理 | ✅ 正常推理 | ⏭️ 跳过，返回空 |
| 召回（recall） | ✅ 正常 | ✅ 正常（不依赖 LLM） |
| 工作记忆 | ✅ 正常 | ✅ 正常（不依赖 LLM） |
| 统计/搜索 | ✅ 正常 | ✅ 正常（不依赖 LLM） |

#### `src/extractor/extract.ts` — Extractor 接受 null LLM

| 改动 | 说明 |
|------|------|
| `constructor(private llm: CompleteFn)` → `constructor(private llm: CompleteFn \| null)` | 允许 null |
| `extract()` 方法新增 `if (!this.llm)` 守卫 | 返回 `{ nodes: [], edges: [] }` + warn 日志 |

#### `src/fusion/analyzer.ts` — runFusion 接受 null LLM

| 改动 | 说明 |
|------|------|
| `llm: CompleteFn` → `llm: CompleteFn \| null` | 允许 null |
| `decideFusion` 调用新增 null 分支 | 无 LLM 时：≥0.95 自动合并，≥0.85 自动链接，其余跳过 |
