# 任务 1.5：修复版本号不一致

**来源**: M2-P1-2  
**执行日期**: 2026-05-25  
**执行人**: AI  
**状态**: ✅ 已完成

---

## 问题描述

项目存在三处版本号引用不一致：

| 文件 | 修复前 | 修复后 |
|------|--------|--------|
| `package.json` | `"2.0.0"` | `"2.0.0"`（不变，单一事实来源） |
| `openclaw-register.ts:37` | `'1.8.0'` | `'2.0.0'` |
| `openclaw-wrapper.ts:198` | `'1.0.0'` | `'2.0.0'` |
| `src/engine/context.ts:453` | `'2.0.0'` | `'2.0.0'`（已正确，不变） |

**影响**：
- 插件管理界面会显示过期版本号
- 版本依赖检查可能使用错误的版本号
- 用户报告 bug 时使用错误的版本号

## 执行内容

### 1. 修改 `openclaw-register.ts:37`

```diff
- version: '1.8.0',
+ version: '2.0.0',
```

### 2. 修改 `openclaw-wrapper.ts:198`

```diff
- version: '1.0.0',
+ version: '2.0.0',
```

### 3. 新增版本一致性检查脚本

在 `package.json` 中添加 `check:versions` 脚本，自动检查三个文件中硬编码的版本号是否与 `package.json` 一致：

```
check:versions: 扫描 openclaw-register.ts, openclaw-wrapper.ts, src/engine/context.ts
                 中的 version: 'X.Y.Z' 是否全等于 package.json version
```

### 4. 验证

```bash
npm run check:versions
# 输出：✅ All version references match package.json (2.0.0)
```

三次版本引用行确认：
```
openclaw-register.ts:37:   version: '2.0.0',
openclaw-wrapper.ts:198:   version: '2.0.0',
src/engine/context.ts:453: version: '2.0.0',
package.json:               "2.0.0"
```

---

## 验证结论

✅ 三处硬编码版本号均已统一为 `'2.0.0'`，与 `package.json` 一致。新增 `check:versions` 脚本可在后续版本升级时自动检测不一致。

---

## 说明

为何不采用 `import pkg from '../package.json'` 动态读取？
- 该项目 ESM 编译后从 `dist/` 加载，TS 编译器不调整 JSON import 路径
- `package.json` 虽在 npm `files` 中，但导入路径需从编译产物倒推，与源码路径不同
- 采用**硬编码 + CI 检查**方案，简单可靠，`check:versions` 脚本防止偏离
