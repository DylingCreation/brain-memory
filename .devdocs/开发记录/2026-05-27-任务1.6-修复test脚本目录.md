## 问题描述

`package.json` 中 4 个 test 脚本 (`test:unit`, `test:integration`, `test:performance`, `test:e2e`) 指向 4 个不存在的子目录。71 个测试文件平铺在 `test/` 根目录下，脚本运行时静默返回 0 tests（vitest 旧版表现）或退出码 1（新版表现），但无论如何都不运行任何实际测试。

## 执行内容

### 1. 创建 4 个测试子目录

```bash
mkdir -p test/unit test/integration test/performance test/e2e
```

### 2. 分类移动 71 个测试文件

| 目标目录 | 数量 | 分类依据 |
|----------|------|----------|
| `test/unit/` | 18 | 纯逻辑/工具函数测试（decay, heuristic, truncate, json, noise, pagerank, scope, utils 等） |
| `test/integration/` | 47 | 需数据库/外部 API/多模块协作（store, extractor, recaller, integration, context-engine 等） |
| `test/performance/` | 6 | 性能基准测试（perf-*, c7-recall-baseline） |
| `test/e2e/` | 0 | 暂无可放入的端到端测试（创建 README 占位） |

使用 `git mv` 保留文件历史。

### 3. 修复 import 路径

移动后所有测试文件加深一级目录，需修复相对 import：
- `../src/` → `../../src/`（源文件导入）
- `./helpers` → `../helpers`（test helper 导入）

使用 sed 批量处理单引号和双引号两种格式。

### 4. 创建占位 README

- `test/e2e/README.md` — 端到端测试说明
- `test/performance/README.md` — 性能基准测试说明
