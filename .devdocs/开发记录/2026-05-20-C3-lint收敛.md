### 根因诊断

按照 METHODOLOGY.md §5.8（Lint 问题架构级分析方法）逐文件分析后，发现所有 122 warnings 的根因相同：

**ESLint 核心 `no-unused-vars` 规则不理解 TypeScript 语义：**
- TypeScript 构造函数 `private param` 被存储为 `this.param` 类属性，核心规则不追踪这种间接使用 → 误报
- TypeScript 类型别名参数名（如 `EmbedFn = (text: string) => ...`）纯粹用于文档说明 → 误报  
- TypeScript 接口方法参数（如 `IStorageAdapter` 的 60+ 方法签名）必须匹配接口契约 → 误报

而 `@typescript-eslint/no-unused-vars`（已在配置中）正确理解以上所有 TypeScript 语义。

### 修复方式

**单行配置变更**：关闭核心 `no-unused-vars`，仅保留 `@typescript-eslint/no-unused-vars`。

```diff
- "no-unused-vars": "warn",
+ "no-unused-vars": "off",
```
