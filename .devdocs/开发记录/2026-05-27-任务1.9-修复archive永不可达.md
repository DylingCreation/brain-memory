## 问题描述

`src/session/compressor.ts:57-62` 的 `evaluateSessionValue` 函数中，`archive` 建议永远无法被返回。

### 原因

条件判断顺序错误：

```typescript
// 修复前（错误）
if (valueScore < 0.2 && messages > 20) {
    compressRecommendation = 'compress';   // ← 先捕获了所有 valueScore < 0.2 的情况
} else if (valueScore < 0.1 && messages > 50) {
    compressRecommendation = 'archive';    // ← 这是第一个条件的子集，else if 中永不可达
}
```

`valueScore < 0.1` 是 `valueScore < 0.2` 的**严格子集**。当 `valueScore = 0.05` 时，先被 `if (valueScore < 0.2)` 捕获 → `compress`。`else if` 永远不会被求值。

### 影响

任何会话的 `archive` 操作永远不会被触发。长期运行的服务中，低价值会话会无限积累而不被归档。

---

## 执行内容

### 修复

```typescript
// 修复后（正确）
// 更严格的条件放在前面，确保先匹配 archive（需要更低分数 + 更多消息）
if (valueScore < 0.1 && messages > 50) {
    compressRecommendation = 'archive';
} else if (valueScore < 0.2 && messages > 20) {
    compressRecommendation = 'compress';
} else {
    compressRecommendation = 'keep';
}
```
