## 一、开发内容

### 1. 修改文件

#### `src/engine/context.ts` — 增强 `getStats()`

**旧版**（仅 3 个基础指标）：
```typescript
getStats(): { nodeCount: number; edgeCount: number; sessionCount: number }
```

**新版**（全维度统计，新增 `EngineStats` 接口）：

| 字段 | 说明 |
|------|------|
| `nodeCount` / `nodes.total` | 节点总数（向后兼容 + 嵌套详细） |
| `nodes.active` | 活跃节点数 |
| `nodes.deprecated` | 废弃节点数 |
| `nodes.byType.{task,skill,event}` | 按类型分类 |
| `nodes.byTemporalType.{static,dynamic}` | 按时态分类 |
| `nodes.bySource.{user,assistant}` | 按来源分类 |
| `edgeCount` / `edges.total` | 边总数 |
| `communities` | 社区数 |
| `vectors` | 向量化节点数 |
| `sessionCount` | 会话数 |
| `dbSizeBytes` | 数据库文件大小 |
| `schemaVersion` | Schema 版本号 |
| `uptimeMs` | 引擎运行时长 |
| `embedCache.{size,hits,misses,hitRate}` | 嵌入缓存统计 |
| `queryTimeMs` | 本次查询耗时 |

**设计原则**：
- 向后兼容：顶层 `nodeCount` / `edgeCount` / `sessionCount` 保持不变
- 分层结构：`nodes` 对象内按维度细分，便于扩展
- 轻量查询：全部为 SQLite COUNT 聚合，单条查询 < 1ms

#### `src/engine/embed.ts` — 缓存命中率统计

| 组件 | 说明 |
|------|------|
| `cacheHits` / `cacheMisses` | 模块级计数器 |
| `cacheGet()` | 每次缓存命中/未命中更新计数器 |
| `EmbedCacheStats` 接口 | `{ size, hits, misses, hitRate }` |
| `getEmbedCacheStats()` | 公开函数，返回当前缓存统计 |
| `resetEmbedCacheStats()` | 重置计数器（测试/按需使用） |

#### `test/engine-stats.test.ts` — 5 个测试用例

| # | 测试用例 | 验证内容 |
|---|---------|---------|
| 1 | returns complete structure on empty database | 空 DB 返回完整结构，所有字段存在 |
| 2 | query time is under 100ms | 查询耗时 < 100ms（验收标准） |
| 3 | uptime increases between calls | uptimeMs 随时间增长 |
| 4 | multiple calls return consistent structure | 多次调用结构一致 |
| 5 | backward compatibility: top-level matches nested | 顶层字段与嵌套字段一致 |

## 四、使用示例

```typescript
const engine = await createContextEngine(config);
const stats = engine.getStats();

// 快速概览（向后兼容）
console.log(`Nodes: ${stats.nodeCount}, Edges: ${stats.edgeCount}`);

// 详细统计
console.log(`Task nodes: ${stats.nodes.byType.task}`);
console.log(`Active: ${stats.nodes.active}, Deprecated: ${stats.nodes.deprecated}`);
console.log(`Vectors: ${stats.vectors}, Communities: ${stats.communities}`);
console.log(`Cache hit rate: ${(stats.embedCache.hitRate * 100).toFixed(1)}%`);
console.log(`Query took: ${stats.queryTimeMs}ms`);
```
