## 一、开发内容

### 核心问题

v0.2.0 的 OpenClaw 插件使用 `api.on()` 注册钩子，这是非规范的注册方式。同时 Hook Handler 签名 `(event: any, ctx: any)` 与 OpenClaw 真实 `InternalHookEvent` 格式不兼容。

### 1. 修改文件

#### `openclaw-wrapper.ts` — 真实 API 适配

| 改动 | 说明 |
|------|------|
| **新增 `normalizeHookArgs()`** | 统一参数解析函数，同时兼容两种调用格式：① `(InternalHookEvent)` 规范格式 ② `(event, ctx)` 旧格式 |
| **Hook 注册** | 从 `api.on()` 改为优先 `api.registerHook()`，向下兼容 `api.on()` |
| **Handler 签名** | 5 个钩子函数全部从 `(event, ctx)` 改为 `(...args: any[])` + `normalizeHookArgs()` 解析 |
| **变量引用修复** | 所有钩子内部对 `sessionId`、`agentId`、`content` 的引用改为从 `normalizeHookArgs()` 返回值获取 |

#### `normalizeHookArgs()` 设计

```typescript
function normalizeHookArgs(args: any[]): {
  sessionId: string;
  agentId: string;
  workspaceId: string;
  content: string;
  channel?: string;
  rawEvent: any;
}
```

**检测逻辑**：
- 如果 `args[0]` 包含 `sessionKey` 和 `type` 字段 → 判定为 `InternalHookEvent` 格式
- 否则 → 判定为旧 `(event, ctx)` 格式

**InternalHookEvent 映射**：
- `event.sessionKey` → sessionId（优先），fallback 到 `event.context.conversationId`
- `event.context.accountId` → agentId
- `event.context.content` → content
- `event.context.channel` → channel

### 2. 无需修改的文件

| 文件 | 原因 |
|------|------|

| `src/engine/context.ts` | 上下文引擎不受影响 |
| `src/types.ts` | 类型定义不变 |

## 五、向后兼容性

- **双格式兼容**：`normalizeHookArgs()` 同时支持新旧两种调用格式，不会破坏现有 OpenClaw 版本
- **注册方法降级**：优先 `api.registerHook()` → 降级 `api.on()` → 报错，确保兼容性
- **API 签名不变**：导出的函数名和数量不变（`register/init/activate/deactivate/message_received/message_sent/session_start/session_end/before_message_write`）
- **行为不变**：钩子内部逻辑完全不变，仅参数解析方式改变
