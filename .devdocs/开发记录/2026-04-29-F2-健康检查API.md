### 3. `healthCheck()` 设计详情

```
┌─ HealthStatus ─────────────────────────────┐
│ status: "healthy" | "degraded" | "unhealthy"│
│ uptimeMs: number                            │
│ schemaVersion: number                       │
│                                            │
│ components: {                               │
│   database:  { status, detail? }            │
│   llm:       { status, detail? }            │
│   embedding: { status, detail? }            │
│ }                                           │
│                                            │
│ stats?: {                                   │
│   nodeCount, edgeCount, vectorCount,        │
│   communityCount, dbSizeBytes               │
│ }                                           │
└────────────────────────────────────────────┘
```

**状态判定逻辑**：

| 条件 | 整体状态 |
|------|---------|
| DB 异常 | `unhealthy` |
| DB 正常，但 LLM 或 Embedding 未配置 | `degraded` |
| 全部正常 | `healthy` |

**LLM/Embedding 状态说明**：
- `ok` — API key 已配置，客户端可用
- `not_configured` — 无 API key，功能跳过（非错误，是优雅降级）
- `error` — （预留，后续可扩展 API 实际调用验证）

## 四、使用示例

```typescript
import { createContextEngine } from "brain-memory";

const engine = await createContextEngine(config);
const health = engine.healthCheck();

console.log(health.status);           // "degraded" (无 LLM/Embedding)
console.log(health.uptimeMs);         // 12345
console.log(health.schemaVersion);    // 1
console.log(health.components.llm);   // { status: "not_configured", detail: "..." }
console.log(health.stats?.nodeCount); // 42
```

## 五、后续扩展
