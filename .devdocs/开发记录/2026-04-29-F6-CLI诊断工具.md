## 一、开发内容

### 1. 新增文件

#### `scripts/doctor.js` — CLI 诊断核心脚本

**功能**：一键检查环境、依赖、配置、数据库状态，输出结构化诊断报告。

**检查项**：

| 检查组 | 检查项 | 判定标准 |
|--------|--------|---------|
| **环境** | Node.js 版本 | ≥ 20 通过，否则失败 |
| **依赖** | node_modules 存在性 | 不存在则报错 |
| **依赖** | 关键依赖安装状态 | `@photostructure/sqlite` 等核心依赖 |
| **依赖** | SQLite FTS5 扩展 | 内存 DB 测试 FTS5 建表 |
| **配置** | config.js 存在性 | 不存在则警告 |
| **配置** | LLM 配置 | 检查 `config.js` 中的 `LLM_CONFIG`，识别模板默认值 |
| **配置** | Embedding 配置 | 检查 `config.js` 中的 `EMBEDDING_CONFIG`，识别模板默认值 |
| **配置** | 环境变量 | `ANTHROPIC_API_KEY`、`BM_LOG_LEVEL`、`BM_DEBUG`、`BM_LOG_LLM` |
| **数据库** | 文件存在性 | 默认路径 `~/.openclaw/brain-memory.db` |
| **数据库** | 文件大小 | < 100MB 通过，否则警告 |
| **数据库** | WAL/SHM 残留 | 存在则警告（数据库可能未正常关闭） |
| **数据库** | Schema 版本 | 读取 `bm_meta` 表 |
| **数据库** | 表统计 | bm_nodes/bm_edges/bm_vectors/bm_messages/bm_communities 行数 |

**输出格式**：
- ✓（绿色）= 通过
- ⚠（黄色）= 警告（非阻塞，建议关注）
- ✗（红色）= 失败（需修复）

**使用方式**：
```bash
# 方式 1：npm script
npm run doctor

# 方式 2：bin 命令（安装后）
npx brain-memory-doctor

# 方式 3：直接运行
node scripts/doctor.js
```
