# 任务 1.8：修复 tsconfig 路径别名 + tsconfig.test.json

**执行人**: AI  
**状态**: ✅ 已完成

## 问题描述

### 问题 A：`tsconfig.json` 路径别名指向错误目录

```jsonc
// 修复前
"@tests/*": ["./tests/*"]  // tests/ 目录不存在！

// vitest.config.ts 使用（正确）
"@test": "./test"          // test/ 目录存在
```

`tsconfig.json` 用 `tests`（带 s），`vitest.config.ts` 用 `test`（不带 s）。两个别名系统不一致。

### 问题 B：`tsconfig.test.json` 引用不存在目录

```jsonc
// 修复前
"include": ["src/**/*", "tests/**/*", "test/**/*"]
//                       ^^^^^^^^^^^^  目录不存在！

// 修复后
"include": ["src/**/*", "test/**/*"]
```

---

## 执行内容

### 1. 引用检查

在修复前确认所有路径别名的实际使用情况：

| 别名 | 代码中使用 | tsconfig 路径 | 目标存在？ |
|------|-----------|---------------|-----------|
| `@tests/*` | 无引用 | `./tests/*` | ❌ tests/ 不存在 |
| `@test/*` | 无引用（仅 vitest 使用） | — | ✅ test/ 存在 |
| `@src/*` | 无引用 | `./src/*` | ✅ 存在（保留） |
| `@utils/*` | 无引用 | `./src/utils/*` | ✅ 存在（保留） |
| `@types/*` | 无引用 | `./src/types/*` | ✅ 存在（保留） |

### 2. 修正 tsconfig.json 路径别名

```diff
- "@tests/*": ["./tests/*"],
+ "@test/*": ["./test/*"],
```

与 `vitest.config.ts` 中的 `'@test': './test'` 保持一致。

### 3. 修正 tsconfig.test.json include

```diff
  "include": [
    "src/**/*",
-   "tests/**/*",
    "test/**/*"
  ],
```

### 4. 验证

```bash
npx tsc -p tsconfig.test.json --noEmit
```

Exit code: 0。所有类型检查错误均为预存（vitest 全局变量、.ts 扩展名导入），与本次修改无关。

---

## 验证结论

✅ `tsconfig.json` 路径别名从 `@tests/* → ./tests/*` 修正为 `@test/* → ./test/*`，与 `vitest.config.ts` 一致
✅ `tsconfig.test.json` 移除不存在的 `tests/**/*` 引用
✅ 编译验证通过（exit code 0）
