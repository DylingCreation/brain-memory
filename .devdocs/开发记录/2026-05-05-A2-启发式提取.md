## 一、开发内容

### 核心问题

v0.2.0 中 LLM 不可用时，知识提取功能完全跳过（返回空结果）。A-2 通过三级提取架构，实现无 LLM 时仍能产出粗粒度知识，有 LLM 时通过合并提升召回率。

### 1. 新增文件

#### `src/extractor/heuristic.ts` — 启发式提取模块（新文件）

**功能**：纯规则提取，无需 LLM，延迟 < 10ms。

**核心组件**：

| 组件 | 说明 |
|------|------|
| **8 类正则规则** | `HEURISTIC_RULES` 数组，覆盖 profile / preferences / entities / tasks / events / skills / cases / patterns |
| **代码块提取** | `extractCodeBlocks()` — 解析 ```lang\n...\n``` 格式，产出 SKILL 节点 |
| **命令提取** | `extractCommands()` — 匹配 npm / docker / git / curl / kubectl / pip / cargo / go 等 CLI 命令 |
| **主提取函数** | `heuristicExtract(messages, options)` — 遍历消息，去重产出 ExtractionResult |
| **置信度判断** | `heuristicConfidence(result)` — ≥3 节点且 ≥2 类 → high；≥1 节点 → medium；空 → low |

**设计决策**：
- 不提取边关系（启发式无法可靠推断图关系，边提取留给 LLM）
- 节点名称去重：通过 `normalizeName()` 归一化，避免重复
- 内容截断：300 字符限制，防止过长内容
- 代码块 < 5 字符直接跳过（避免无意义片段）

### 2. 修改文件

#### `src/extractor/extract.ts` — 三级决策流

| 改动 | 说明 |
|------|------|
| **新增 import** | `import { heuristicExtract, heuristicConfidence } from "./heuristic"` |
| **新增 mergeResults()** | 合并启发式 + LLM 结果，按归一化名称去重，LLM 结果优先 |
| **重构 extract()** | 从"LLM 不可用时返回空"改为"先跑启发式，再跑 LLM，最后合并" |

**三级决策流**：

```
┌─────────────────────────────────────────────────┐
│ 一级：启发式提取（每次必跑，无 LLM 也生效）         │
│ 8类正则 + 代码块 + 命令 → 产出粗粒度节点            │
│ 延迟 < 10ms                                     │
├─────────────────────────────────────────────────┤
│ 二级：LLM 提取（LLM 可用时运行）                    │
│ 标准 LLM 提取流程（EXTRACT_SYS prompt）              │
├─────────────────────────────────────────────────┤
│ 三级：合并去重（LLM 可用时）                        │
│ LLM 结果优先，补充启发式独有节点                     │
│ LLM 失败/为空时 → 直接返回启发式结果                 │
└─────────────────────────────────────────────────┘
```

**降级效果**：

| 场景 | 旧行为（v0.2.0） | 新行为（A-2） |
|------|-----------------|--------------|
| LLM 不可用 | 返回空，零产出 | 启发式产出粗粒度节点 |
| LLM 可用且成功 | LLM 提取 | LLM + 启发式合并（召回率提升） |
| LLM 可用但失败 | 容错提取 → 可能空 | LLM 失败 → 启发式兜底 |

## 四、向后兼容性

- **API 签名不变**：`Extractor.extract()` 返回值类型不变（ExtractionResult）
- **行为增强**：LLM 不可用时从"返回空"变为"返回启发式结果"，属于功能增强，非破坏性变更
- **LLM 可用时**：节点数可能增加（合并结果），但 LLM 原有节点不变，只是补充
- **集成测试适配**：断言从 `toBe(N)` 改为 `toBeGreaterThanOrEqual(N)`，反映合并行为
