## 一、开发内容

### 1. 新增文件

#### `.devdocs/技术决策/2026-04-29-SQL参数化审计报告.md` — 安全审计文档

**审计范围**：`src/` 目录下所有 `.ts` 文件中的 SQL 查询。

**审计方法**：
1. `grep -rn "\.prepare\|\.exec" src/` 提取所有 SQL 调用
2. 逐文件审查每个 SQL 查询的构造方式
3. 验证 `?` 占位符与参数绑定匹配
4. 检查模板字符串拼接，确认无用户输入直接进入 SQL
5. 验证 `db.exec()` 仅用于 DDL/事务控制

**审计结果**：

| 指标 | 数据 |
|------|------|
| `.prepare()` 调用 | 89 处，全部参数化 |
| `.exec()` 调用 | 8 处，全部为 DDL/事务 |
| 涉及文件 | 8 个 |
| 发现漏洞 | 0 |
| 总体判定 | ✅ 100% 参数化，安全 |

**重点验证的 3 种动态 SQL 构造模式**：

| 模式 | 位置 | 安全性 | 说明 |
|------|------|--------|------|
| Scope 子句拼接 | `src/scope/isolation.ts` | ✅ 安全 | 仅生成 `?` 占位符结构，值通过 params 传递 |
| IN 占位符生成 | `vectorSearchWithScore`、`graphWalk` | ✅ 安全 | 由 `ids.map(() => "?").join(",")` 生成，仅含 `?` |
| FTS5 MATCH | `searchNodes` | ✅ 安全 | MATCH 参数通过 `.all(ftsQuery)` 绑定，非 SQL 拼接 |
