## 一、开发内容

### 1. 新增文件

#### `src/store/migrate.ts` — 数据库迁移核心模块

| 组件 | 说明 |
|------|------|
| `CURRENT_SCHEMA_VERSION` | 当前 schema 版本号常量（= 1） |
| `getSchemaVersion(db)` | 读取数据库当前 schema 版本号，bm_meta 不存在时返回 0 |
| `migrate(db)` | 执行所有待执行的迁移，幂等操作 |

**核心设计**：
- 新建 `bm_meta` 表：`CREATE TABLE IF NOT EXISTS bm_meta (key TEXT PRIMARY KEY, value TEXT NOT NULL)`
- `migrate()` 三步走：① 确保 bm_meta 存在 → ② 读取当前版本 → ③ 版本为 0 时初始化为 1，之后按版本号递进执行迁移
- 已预留 `migrateTo_v2` 等迁移函数的扩展模板
- 幂等保证：重复调用 `migrate()` 不会改变版本号，也不会重复执行迁移

### 2. 修改文件

#### `src/store/db.ts` — 迁移集成

| 改动 | 说明 |
|------|------|
| 新增 `import { migrate } from "./migrate"` | 引入迁移模块 |
| `initDb()` 末尾调用 `migrate(db)` | 每次初始化 DB 后自动执行迁移，确保版本对齐 |

## 四、后续扩展

未来 schema 变更时，只需：
1. 在 `migrate.ts` 中新增 `migrateTo_vN(db)` 函数
2. 在 `migrate()` 中添加版本判断：`if (version < N) { migrateTo_vN(db); version = N; }`
3. 更新 `CURRENT_SCHEMA_VERSION = N`
4. 在 `migrate.test.ts` 中新增对应的迁移测试
