## 一、调研内容

### B-1 记忆注入格式 overhaul

**研读文件**：`src/format/assemble.ts`、`openclaw-wrapper.ts`（before_message_write 钩子）、`src/types.ts`（BmConfig）

**核心发现**：
1. `assembleContext` 已有完整能力（token 预算、三种策略、语言感知估算、社区分组、情景记忆），但**未被 OpenClaw 链路调用**
2. 当前 `before_message_write` 钩子直接把原始 `memoryContext`（JSON 对象）附加到事件，未经格式化
3. `tokenBudget` 参数已存在但**没有调用方计算预算值**

**方案**：接入 `assembleContext` + 新增 `memoryInjection` 配置项（strategy/tokenBudget/maxNodes）

---

### B-2 多 Agent 记忆共享策略

**研读文件**：`src/scope/isolation.ts`、`src/engine/context.ts`（recall 方法）

**核心发现**：
1. `ScopeFilter.allowCrossScope` 字段已声明但**完全未使用**
2. 当前跨范围 recall 是硬编码 fallback（无结果 → 全局查询），非配置化
3. `bm_nodes.scope_agent` 字段已存在，数据结构就绪

**方案**：新增 `memorySharing` 配置（isolated/mixed/shared 三模式 + sharedCategories 控制）

---

### B-3 遗忘曲线默认开启

**研读文件**：`src/decay/engine.ts`、`src/types.ts`（DecayConfig + DEFAULT_CONFIG）

**核心发现**：
1. Weibull 衰减引擎已成熟（三层分层、动态加速、复合评分、召回时衰减）
2. `decay.enabled` 默认 false，需改为 true
3. 节点有 `status` 字段但**无自动归档逻辑**
4. 软标记方案（deprecated）即可满足需求，recall 已过滤 status='active'

**方案**：改默认值 + runMaintenance() 增加归档逻辑 + CLI 命令（decay-stats/promote/demote）
