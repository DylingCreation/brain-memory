## 一、开发内容

### 核心问题
`ScopeFilter.allowCrossScope` 已声明但完全未使用，跨 Agent 共享是硬编码 fallback。

### 1. 新增类型（src/types.ts）

```typescript
export type SharingMode = "isolated" | "mixed" | "shared";

export interface MemorySharingConfig {
  enabled: boolean;
  mode: SharingMode;
  sharedCategories: MemoryCategory[];  // default ["profile", "preferences"]
  allowedAgents: string[];             // default []（空=所有 Agent）
}
```

### 2. 修改文件

#### src/scope/isolation.ts — buildScopeFilterClause 扩展

| 改动 | 说明 |
|------|------|
| ScopeFilter 新增字段 | `sharingMode`, `sharedCategories`, `currentAgentId`, `allowedAgents` |
| 新增 sharingMode 逻辑 | mixed 模式按 sharedCategories 过滤，shared 模式无限制，isolated 模式完全隔离 |
| 新增 allowedAgents 过滤 | 指定 Agent 列表时可限制共享范围 |

#### src/engine/context.ts — recall 方法重构

| 改动 | 说明 |
|------|------|
| 替换硬编码 fallback | 使用 `memorySharing` 配置驱动跨范围召回 |
| 保留向后兼容 | 配置不存在时默认 mixed + profile/preferences |

### 3. 三种共享模式

| 模式 | 行为 |
|------|------|
| `isolated` | 完全隔离，各 Agent 只能读取自己的记忆 |
| `mixed` | 仅 `sharedCategories` 中的类别可跨 Agent（默认 profile + preferences） |
| `shared` | 所有类别完全共享 |
