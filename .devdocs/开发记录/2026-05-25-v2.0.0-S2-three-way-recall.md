## 一、目标

将双路径召回（精确 + 泛化）扩展为四路并行：
- ① SQLite FTS5 关键词（已有）
- ② SQLite 社区向量（已有）
- ③ LanceDB 语义搜索（新增）
- ④ Agent 长期记忆 externalNodes（新增）

## 二、设计

### ISearchIndex 注入

```
ContextEngine
  ├── storage: IStorageAdapter      ← SQLite 真值
  ├── searchIndex: ISearchIndex     ← LanceDB 伴随索引 (新增)
  └── recaller: Recaller            ← 四路召回
```

- `Recaller` 构造接受可选的 `ISearchIndex`
- `setSearchIndex()` 方法支持延迟注入
- `ContextEngine.setSearchIndex()` 透传给 Recaller

### 四路种子合并

```
精确路径 (FTS5+Vector) ─┐
泛化路径 (Community)     ├→ unifiedSeeds → graphWalk → PPR → decay → result
语义路径 (LanceDB)       │
外部节点 (externalNodes) ┘
```

语义搜索通过 `_getSemanticSeeds()` 实现：embed(query) → searchIndex.semanticSearch() → nodeIds。只用 nodeId，真值从 SQLite 查——保证 SQLite 始终是权威源。

### externalNodes 参数

供 OpenClaw 插件层注入 Agent 的 USER/MEMORY 文件中的长期记忆。插件读取 Agent 文件 → 构建 BmNode[] → 传入 `recall(query, scopeFilter, sourceFilter, externalNodes)`。
