## 一、开发内容

### 核心问题
OpenClaw 的 `before_message_write` 钩子直接把原始 JSON 附加到事件，未经 `assembleContext` 格式化，缺少 token 预算控制。

### 1. 新增类型（src/types.ts）

```typescript
export interface MemoryInjectionConfig {
  enabled: boolean;
  strategy: "full" | "summary" | "adaptive" | "off";
  tokenBudget: number;      // default 6000
  maxNodes: number;         // default 12
  includeEpisodic: boolean; // default true
}
```

### 2. 修改文件

#### src/plugin/core.ts — getMemoryContext 重构

| 改动 | 说明 |
|------|------|
| 新增 import | `assembleContext`, `allActiveNodes`, `allEdges` |
| 重构 getMemoryContext | 新增 `memoryInjection` 配置读取，调用 `assembleContext` 格式化 recalled 节点 |
| 新增 _getRawMemoryContext | 保留旧行为作为 fallback |
| 重构 beforeMessageSend | 调用 getMemoryContext，附加 `formattedMemory`（xml/systemPrompt/episodicXml） |

#### src/types.ts — 新增配置项

| 改动 | 说明 |
|------|------|
| 新增 `MemoryInjectionConfig` 接口 | B-1 注入格式化配置 |
| `DEFAULT_CONFIG` 增加 `memoryInjection` | 默认 adaptive 策略，6000 token 预算，12 节点上限 |

### 3. 格式化输出结构

```typescript
{
  memoryContext: string;          // 工作记忆上下文
  relatedNodes: BmNode[];         // 召回节点列表
  tokenEstimate: number;          // 估算 token 数
  formattedXml: string | null;    // XML 格式知识图谱
  systemPrompt: string;           // 系统提示补充
  episodicXml: string;            // 情景记忆（原始对话片段）
}
```
