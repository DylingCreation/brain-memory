## 问题描述

任务 1.9 在 2026-05-25 初次执行时修复了 `evaluateSessionValue` 的条件顺序（archive `<0.1` 先于 compress `<0.2`），逻辑正确。但修复方案要求的 3 个专项回归测试**缺失**——仅有 `toContain(['keep','compress','archive'])` 宽泛断言，无法检测 archive 分支是否可达。

## 执行内容

在 `test/integration/compressor-coverage.test.ts` 的 `evaluateSessionValue — boundary coverage` describe 块中新增 2 个测试：

### 测试 1：archive 推荐

```typescript
it('recommends archive for very low value sessions with many messages', () => {
  // 55 messages, 0 knowledge nodes → valueScore < 0.1
  const result = evaluateSessionValue(db, 'junk-session');
  expect(result.valueScore).toBeLessThan(0.1);
  expect(result.compressRecommendation).toBe('archive');
});
```

### 测试 2：archive 优先于 compress

```typescript
it('archive takes priority over compress when condition is met', () => {
  // 60 messages, 1 knowledge node → 同时满足 archive 和 compress 条件
  const result = evaluateSessionValue(db, 'priority-session');
  expect(result.compressRecommendation).toBe('archive');
  expect(result.compressRecommendation).not.toBe('compress');
});
```
