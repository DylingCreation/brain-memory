## 四、关键技术决策

| # | 决策 | 结论 | 理由 |
|---|------|------|------|
| 1 | 脏标记位置 | 放在接口内 | 缓存失效可复用；不同后端实现不同 |
| 2 | 图结构加载 | `loadGraphStructure()` | PageRank/LPA 纯算法不变，仅输入来源改为接口 |
| 3 | `graphWalk` 位置 | 留在接口内 | 不同后端实现不同（SQL CTE vs 内存遍历） |
| 4 | `getDb()` 桥接 | 保留 | `assembleContext` 等遗留代码尚未迁移 |

## 五、遇到的问题与解决

| 问题 | 根因 | 解决 |
|------|------|------|
| 69 测试失败 | 测试直接传 `db` 给构造器 | 新增 `createTestStorage()` 辅助函数 |
| `SQLiteStorageAdapter is not a constructor` | `helpers.ts` 从 `adapter.ts` import | 改为 import `sqlite-adapter.ts` |
| Pagerank 单节点测试失败 | `loadGraph()` 30s TTL 缓存跨测试污染 | beforeEach 调用 `invalidateGraphCache()` |
| llm-integration INSERT 列数不匹配 | 测试 SQL 缺少 `source` 列 | 补上 `'user'` 值 |
| llm.test.ts Unhandled Rejection | vitest fake timer + async retry | `promise.catch(() => {})` 抑制 |
| LLM 集成测试 401 认证失败 | BaseURL 为默认而非用户指定 | `.env` 改用 `coding.dashscope.aliyuncs.com/v1` |
