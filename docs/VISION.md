# brain-memory 愿景

> "让 AI 拥有像人类一样的大脑和记忆"

---

## 一、为什么做这个项目

graph-memory 和 memory-lancedb-pro 各自做到了极致：
- graph-memory 把**知识图谱 + 图算法**做到了极致——PPR 按查询动态排名、社区检测自动分组、情景追溯保留原始对话
- memory-lancedb-pro 把**记忆治理**做到了极致——Weibull 衰减、意图分析、准入控制、偏好槽位、时间感知、Reflection 体系

但它们各走了一条路。graph-memory 没有衰减和治理，memory-lancedb-pro 没有图结构和关系推理。

brain-memory 的目标是**把两条路合起来，做出一个更接近人类大脑的记忆系统**。

---

## 二、人类记忆给我们的启发

### 2.1 多重记忆系统

人脑不是单一记忆，是分工协作的：

| 人类记忆类型 | 功能 | brain-memory 对应 |
|-------------|------|------------------|
| 情景记忆 (Episodic) | 具体场景和经历 | 情景上下文 (episodic traces) |
| 语义记忆 (Semantic) | 抽象知识和概念 | 知识图谱节点 + 向量记忆 |
| 程序记忆 (Procedural) | 技能和操作模式 | SKILL 节点 + 反思提炼 |
| 工作记忆 (Working) | 当前关注点，短期缓存 | WorkingMemory 管理器（✅ 4.4 已完成） |

### 2.2 记忆需要巩固

人的记忆不是录完就存好了。海马体会在睡眠中回放白天经历，把短期记忆转成长期记忆，建立新连接，丢弃不重要的。

**对应设计：** graph-memory 每 7 轮做 PageRank + 社区检测 = 初步的"巩固"。brain-memory 在此基础上增加**反思系统**——会话结束时 LLM 分析整个会话，提取用户画像 delta、Agent 行为教训、持久决策，存为图谱节点（带边连接），参与 PPR 排名和社区检测。

### 2.3 遗忘是功能，不是缺陷

人主动遗忘不重要的东西。memory-lancedb-pro 的 Weibull 衰减模型抓住了这一点——按重要性分层衰退。但重要性不应该只看访问频率和时间，还应该考虑：
- **关联密度**：越核心的知识越不容易忘
- **独特性**：罕见经历记得更牢
- **情感强度**：用户强烈纠正过的错误记得最深

### 2.4 检索是重建，不是回放

人回忆是**根据线索重建**，不是读文件。每次回忆还会**改写**记忆本身。

**对应设计：** PPR 按查询动态排序已经很聪明。brain-memory 增加**推理检索**——对召回结果做图遍历推理（路径分析、隐含关系推导、多跳结论合成），不只是找到最匹配的节点，而是用相关记忆推理出新结论。

### 2.5 记忆能产生新知识

这是最关键的一点。人不是被动记忆，而是会从已有记忆中**推导出新东西**。

这需要一个**推理/反思层**——不仅仅是找到相关记忆，而是用相关记忆产生新洞察。没有反思，只是一个带检索的笔记本。

**brain-memory 的独特优势：** 反思结果不是扁平文本列表（memory-lancedb-pro 的方式），而是**存为图谱节点**——有边连接、能被 PPR 排名、能被社区检测归类、能被衰减模型治理。这让反思从"又多了一堆文本"变成"主动理解对话意义"。

## 三、三层架构设计

```
┌──────────────────────────────────────────────────┐
│               认知层（Cognitive）                  │
│  反思系统 ──→ 图谱节点（带边连接）                 │
│  知识融合 ──→ 发现跨时期本质关联                  │
│  推理检索 ──→ 多跳结论合成                        │
│  工作记忆 ──→ 当前关注点实时维护                  │
│  ← 区分"记忆库"和"大脑"的关键                      │
├──────────────────────────────────────────────────┤
│               管理层（Governance）                 │
│  衰减 / 噪声过滤 / 准入控制 / 意图分析             │
│  时间分类 / 偏好槽位 / Scope 隔离 / 会话压缩       │
├──────────────────────────────────────────────────┤
│               基础层（Foundation）                 │
│  图谱 + 向量 + 提取 + 召回 + 组装 + 维护           │
│  ← 阶段零~三已完成                                │
└──────────────────────────────────────────────────┘
```

### 基础层

知识的基础设施——怎么存、怎么取、怎么组织。

- **图谱引擎**：节点/边的存储、PPR 排名、社区检测、向量去重、维护流水线
- **向量引擎**：Embedding、向量搜索、BM25 全文检索、RRF 融合
- **提取引擎**：从对话中提取结构化知识
- **召回引擎**：精确路径 + 泛化路径 + 混合路径
- **组装引擎**：把召回的记忆组装成 Agent 可用的上下文

### 管理层

记忆的治理——什么该记、什么不该记、什么时候该忘。

- **衰减模型**：Weibull 分层衰减，按重要性、频率、时间综合计算
- **噪声过滤**：过滤问候语、确认语、过短内容等低信息量消息
- **准入控制**：控制记忆写入频率，防止重复和低质量记忆涌入
- **意图分析**：按查询意图分类，影响召回策略
- **时间分类**：区分静态事实 vs 动态信息，影响衰减策略
- **偏好槽位**：结构化提取用户偏好
- **Scope 隔离**：按 session/agent/workspace 隔离记忆
- **会话压缩**：长会话价值评估 + 压缩

### 认知层

记忆的智能——从记忆中推理、反思、产生新知识。brain-memory 的独特设计：反思结果存为图谱节点（不是扁平文本），参与 PPR 排名、社区检测和衰减治理。

- **反思系统**：两类触发（轮次轻量 + 会话重量），LLM 分析对话提取洞察，存为图谱节点（带边连接），安全过滤防止注入攻击
- **知识融合**：发现不同时期类似问题的本质关联，合并碎片化知识
- **推理检索**：对召回结果做图遍历推理，多跳结论合成
- **工作记忆**：实时维护当前任务目标和关注点，注入上下文顶部

## 五、核心原则

1. **提取是天花板** — 垃圾进，垃圾出。好的提取不是"做笔记"，而是"消化"：理解意图、判断价值、建立连接、抽象提升
2. **先做实基础层，再往上叠** — 一口气做三层，大概率三层都做不好
3. **遗忘是功能** — 记忆系统必须有衰减机制，否则会越来越大、越来越慢、越来越不精准
4. **关系比内容重要** — 图谱的价值不在节点本身，在于节点之间的关系。关系让记忆从"笔记"变成"知识"
5. **反思是主动理解，不是被动记录** — 系统不只是记住对话内容，而是理解对话的意义（哪些是重要偏好、哪些是教训、哪些是规律）
6. **反思结果存为图谱节点** — 利用图结构优势，反思结果有边连接、能被 PPR 排名、能被社区检测归类、能被衰减治理
7. **安全第一** — 反思内容会注入到未来对话上下文中，必须过滤 unsafe injectable 内容，防止 prompt injection
8. **成本意识** — 认知层的 LLM 调用必须可控，规则优先于 LLM（如轮次反思用规则打分而非 LLM），阈值触发（如知识融合/推理检索需图谱达到规模）
9. **测试驱动** — 单元测试覆盖纯逻辑，LLM 集成测试验证端到端 Prompt 效果（DashScope qwen3.6-plus，187 个测试全部通过：180 个单元测试 + 7 个 LLM 集成测试）
